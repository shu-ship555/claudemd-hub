# Claude Config Manager - 実装計画書

## 最終目標
Claude Codeの設定ファイル管理サービスを完成させる

---

## 推奨実装順序

### **Phase 1: インフラ & 基盤設定** (優先度: 🔴 高)

#### 1.1 本番環境での本番DB使用
- **説明**: 環境ごとに異なるSupabaseプロジェクトを切り替える仕組みを構築
- **目的**: 開発時に本番DBを破壊しないようにする
- **実装内容**:
  - `.env.local` (開発用)
  - `.env.production` (本番用)
  - 環境に応じたSupabase設定の切り替え
  - Vercelデプロイ時の環境変数設定
- **完了条件**: 本番環境と開発環境を完全に分離できている

---

### **Phase 2: コア機能実装** (優先度: 🔴 高)

#### 2.1 Config ファイル管理機能 (CRUD)
- **説明**: ダッシュボードのメイン機能
- **実装内容**:
  - ファイルアップロード機能
  - ファイル一覧表示
  - ファイル詳細表示
  - ファイル編集機能
  - ファイル削除機能
  - バージョン履歴管理
- **Supabase テーブル**:
  - `config_files` テーブルの RLS 設定
  - ユーザーごとのファイル所有権管理
- **完了条件**: CRUD操作がすべて正常に動作

---

### **Phase 3: UX改善** (優先度: 🟡 中)

#### 3.1 UIのブラッシュアップ
- **説明**: デザインと使いやすさの改善
- **実装内容**:
  - ダッシュボードレイアウト調整
  - Card コンポーネントの活用
  - エラーハンドリング画面
  - ローディング状態の表示
  - 空状態（Empty State）の改善
- **完了条件**: UI/UXが洗練されている

#### 3.2 日本語ローカライズ + 言語切り替え
- **説明**: UIを日本語対応し、言語切り替え機能を実装
- **実装内容**:
  - i18n ライブラリの導入 (next-intl, react-i18next など)
  - 翻訳ファイルの作成 (ja.json, en.json)
  - 言語選択UI（ヘッダーまたは設定）
  - LocalStorage での言語設定の保存
  - URLベースの言語ルーティング（例: `/ja/dashboard`, `/en/dashboard`）
- **対象テキスト**:
  - ボタンラベル
  - フォームラベル
  - エラーメッセージ
  - ページタイトル
- **完了条件**: 日本語と英語を切り替えられる

#### 3.3 ダークモード対応
- **説明**: 既にdark クラスは有効だが、ブラッシュアップ
- **実装内容**:
  - next-themes との統合確認
  - 全ページでのダークモード対応確認
  - カラースキーム（light/dark）の調整
  - システムテーマ自動検出
- **完了条件**: ダークモード/ライトモードの切り替えがスムーズ

---

### **Phase 4: スケール対応** (優先度: 🟢 低)

  - 上限超過時のエラーハンドリング
  - プレミアムプラン案内
  - ウィッシュリスト機能（無制限アクセスを待つリスト）
- **Supabase RLS ポリシー**:
  - ユーザー数チェック
  - ウィッシュリスト登録機能
- **完了条件**: スケーラビリティに対応している
